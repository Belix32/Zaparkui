## Введение

Данный документ представляет собой детализированный план реализации мобильного приложения «Запаркуй» с использованием фреймворка Capacitor. План учитывает специфику российского рынка, включая доступные платёжные системы, картографические сервисы и особенности публикации в российском сегменте App Store и Google Play.

Основные допущения: MVP ориентирован на рынок Москвы, разработка ведётся силами существующей команды с привлечением подрядчиков по необходимости, старт разработки — май 2026 года. Документ определяет пять основных фаз с конкретными задачами, ответственными и критериями приёмки.

Фреймворк Capacitor выбран по причине максимального переиспользования существующего кода React-приложения (90–95%), минимальных сроков разработки MVP (8–10 недель) и возможности поддержки силами текущей команды без найма отдельных мобильных разработчиков.

## Фаза 3: Бронирование и оплата

**Продолжительность:** 4 недели (29 июня – 26 июля 2026 года)  
**Дата начала:** 29 июня 2026 года  
**Дата окончания:** 26 июля 2026 года

### Описание

Фаза реализации бронирования и оплаты включает создание полного цикла бронирования парковки с выбором даты, периода аренды и интеграцией с российской платёжной системой ЮKassa. Это ключевой функционал для монетизации приложения.

Результатом фазы является готовый к использованию цикл бронирования-оплаты с поддержкой российских платёжных методов.

### Задачи

#### Задача 3.1: Экран выбора даты и периода

**Описание:** Компонент выбора периода аренды с календарём и выбором времени.

**Содержание работ:**

- Компонент DateRangePicker (календарь)
- Выбор даты начала и окончания аренды
- Выбор времени въезда/выезда (почасово)
- Расчёт стоимости на основе периода
- Валидация выбранных дат (не в прошлом)
- Подтверждение доступности (API проверка)

**Ответственный:** react-specialist  
**Зависимости:** Задача 2.4  
**Срок:** 4 дня (29 июня – 2 июля)  
**Критерии приёмки:**

- Календарь работает корректно
- Стоимость рассчитывается верно
- Валидация дат работает

#### Задача 3.2: Процесс бронирования

**Описание:** Создание заявки на бронирование, выбор места (при наличии), подтверждение.

**Содержание работ:**

- Форма бронирования (данные арендатора)
- Выбор конкретного места (при multiple slots)
- Резюме бронирования (сумма, даты, парковка)
- Подтверждение бронирования
- Номер бронирования (UUID)
- Уведомление о создании бронирования

**Ответственный:** react-specialist  
**Зависимости:** Задача 3.1  
**Срок:** 4 дня (3–6 июля)  
**Критерии приёмки:**

- Бронирование создаётся успешно
- Номер бронирования генерируется
- Данные сохраняются корректно

#### Задача 3.3: Интеграция ЮKassa

**Описание:** Интеграция платёжного шлюза ЮKassa для приёма платежей от российских пользователей.

**Содержание работ:**

- Настройка SDK ЮKassa для Capacitor
- Серверная часть (Edge Functions) для создания платежей
- Обработка redirect после оплаты
- Методы оплаты: карты (Visa, MasterCard, МИР), СберPay, Тинькофф Pay
- Обработка webhook от ЮKassa
- Токенизация карт для повторных платежей

**Ответственный:** fullstack-developer  
**Зависимости:** Задача 1.4  
**Срок:** 5 дней (7–11 июля)  
**Критерии приёмки:**

- Платёж создаётся успешно
- Оплата проходит через ЮKassa
- Webhook обрабатывается корректно

#### Задача 3.4: Экран оплаты

**Описание:** Пользовательский интерфейс для выбора метода оплаты и ввода данных карты.

**Содержание работ:**
